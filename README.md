# oh-my-antigravity (OMA) 🚀

> [!IMPORTANT]
> **Gemini CLI is transitioning to Antigravity CLI.** Support for Individual-tier users on Gemini CLI ends on **June 18, 2026**.
> `oh-my-antigravity` (OMA) is the successor to `oh-my-gemini` (OMG), rebuilt from the ground up to leverage the multi-agent power of the Antigravity CLI.

**Supercharge your Antigravity CLI with a teams-first multi-agent orchestration layer.**

## 💡 Origin & Motivation

`oh-my-antigravity` was born because an Antigravity-native version of powerful orchestration tools like [oh-my-claudecode (OMC)](https://github.com/yeachan-heo/oh-my-claudecode) and [oh-my-codex (OMX)](https://github.com/Yeachan-Heo/oh-my-codex) was missing. 

Our mission is to bring that same "supercharged" multi-agent experience to the Antigravity ecosystem, ensuring that Antigravity users have access to the same level of persistence, team-based development, and high-performance workflows.

## 🛡 The OMA Mandate (Core Principles)

- **Feature Parity:** All core features of OMC and OMX serve as the baseline for OMA. While we encourage modifications and additions to leverage Antigravity's unique strengths, **no base feature from OMC/OMX shall be omitted.**
- **Justified Deviations:** If a core feature cannot be implemented due to technical limitations of the Antigravity CLI, a clear technical justification must be documented.
- **Team Orchestration:** Don't work alone. Delegate specialized tasks to a team of agents (Architect, Developer, Verifier).
- **Persistence (The "Ralph" Mentality):** Never give up. Loop through execution and verification until the task is truly done.


## 🛠 Features (Roadmap)

- [ ] **OMA-Ralph:** The persistent manager agent that oversees the entire development lifecycle.
- [ ] **Deep Interview Skill:** A specialized workflow to extract precise requirements.
- [ ] **Standardized Pipeline:** A canonical `Research -> Plan -> Execute -> Verify` workflow.
- [ ] **Durable Memory:** Advanced `.antigravity/memory` management for long-term project context.
