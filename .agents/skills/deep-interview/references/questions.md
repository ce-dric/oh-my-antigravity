# Question Bank for Deep Interview

## 🛠 General Implementation
- What is the primary success metric for this change?
- Are there any existing patterns in the codebase we should follow or avoid?
- What are the known edge cases (e.g., empty state, network failure)?

## 🌐 API & Backend
- What is the expected input/output format (JSON, Protobuf, etc.)?
- Are there specific error codes or handling strategies required?
- Is there a performance or latency constraint?

## 🎨 UI & Frontend
- Is there a specific design system or component library to use?
- How should the feature behave on mobile vs. desktop?
- What are the accessibility requirements (A11y)?

## 🧪 Testing & DevOps
- What level of test coverage is expected (Unit, Integration, E2E)?
- What are the secret-handling requirements (e.g., storage, access controls) and policy constraints? (DO NOT share raw secret contents)
- Should this be behind a feature flag?
